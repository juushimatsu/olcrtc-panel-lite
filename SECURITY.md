# Security policy

Поддерживается последняя опубликованная версия панели и последний успешный upstream bundle.

Для сообщения об уязвимости используйте private GitHub Security Advisory владельца репозитория. Не прикладывайте production credentials, subscription slugs, URI, tokens, private keys или recovery archives.

Практические меры развёртывания описаны в [docs/security.md](docs/security.md).
